02

Passive Enumeration

What's exposed?

Unified chain — commit to muscle memory:

subfinder -silent -d <domain> \
  | dnsx -silent -resp-only \
  | sort -u

# add ASN
subfinder -silent -d <domain> \
  | dnsx -silent -resp-only \
  | asnmap -silent

Fallback order if upstream fails:

  • Mirrors/subfinder/<target>.txt (pre-cached)
  • Mirrors/crtsh/<target>.json (pre-cached)
  • Certspotter · Censys CT · Merklemap

DNS disclosure walk:

dig +short TXT <domain>   # SaaS vendor map
dig +short MX  <domain>   # mail platform
dig +short NS  <domain>   # DNS provider
dig +short SPF <domain>   # email senders

ASN via Cymru DNS — no quota, no key:

# reverse 12.34.56.78 → 78.56.34.12
dig +short TXT 78.56.34.12.origin.asn.cymru.com

Shodan InternetDB (keyless, dedupe to distinct IPs first):

curl -s https://internetdb.shodan.io/<ip> | jq

RFC1918 leak hunt in CT-derived hostnames:

grep -E '(^|[^0-9])(10|172\.(1[6-9]|2[0-9]|3[01])|192\.168)\.' subs.txt

pip install git+https://github.com/laramies/theHarvester.git

recon-ng -w <workspace>
> db insert domains <domain>
> modules load recon/domains-hosts/netcraft
> run
WILDCARD DETECTION FIRST — always before any DNS brute: 
dig +short DEFINITELY-NOT-REAL-XYZQ123.<domain>
NXDOMAIN = safe to brute (on authorized targets). Any IPs returned = wildcard; filter brute output against them.

holehe symbols: [+] hit · [-] confirmed-not-registered · [x] inconclusive (NOT a hit)

Foundation DNS in NS records → gold.foundationdns.com/.net/.org = Cloudflare enterprise.
